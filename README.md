# Claude Instances UI

A browser-native control room for multiple concurrent [Claude Code](https://claude.com/claude-code) sessions. Create sessions, drive them interactively in your browser, and glance at their state from a macOS menu-bar tray — all without leaving the web UI.

Under the hood each session runs inside tmux, so they persist across page refreshes and server restarts. An xterm.js terminal in the browser is a live bidirectional client of the tmux pane. Hooks write per-session state files so observability stays correct even if the server crashes.

---

## Features

- **Create** Claude sessions from the browser (`+ New instance`, pick a cwd, go)
- **Drive** them interactively — full xterm.js terminal, keystrokes and all
- **Watch** status at a glance: running / needs input / idle / ended, colour-coded
- **Rolling summaries** of what each session is doing, generated by an LLM sidecar
- **Groups**, **rename**, **ack** — light session management without tmux keybindings
- **macOS menu-bar tray** (via Hammerspoon) for system-wide awareness
- **External sessions still work** — if you run `claude` in your own terminal (no tmux), it still appears in the list with metadata, just without the in-browser terminal

## Architecture

```
  Browser UI ──► server.py ──► tmux (spawns + owns each `claude`)
     ▲ ▼          │
     │ └──► websocket: xterm.js ◄─► tmux pipe-pane (output) / send-keys (input)
     │
     │            ┌──► state files ◄── hook_writer.py (Claude hooks fire on every event)
     │            │
     └── Hammerspoon menu-bar tray polls /api/instances for glanceable status
```

- **Control plane**: `server.py` (FastAPI). Spawns tmux sessions, routes bidirectional websocket I/O, holds the groups/names/acks state.
- **tmux layer**: each Claude instance runs in its own tmux session on a dedicated socket (`tmux -L ciu`). Session name `ciu-<id>` is correlated with Claude's own session UUID via an env var (`CLAUDE_INSTANCES_UI_OWNED`) that hooks stamp into state files.
- **Observability plane**: Claude hooks write JSON state files at `~/.claude-instances-ui/state/<sid>.json`. The server merges these with tmux session discovery. **This layer works independently of the server** — if you `claude` outside the web UI, you still appear in the dashboard.
- **Frontend**: vanilla JS + xterm.js (CDN). Terminal mounts on session select, unmounts on deselect.
- **Menu bar**: Hammerspoon Lua polls `/api/instances`; slide-in tray renders the compact view.

## Requirements

| Component | Required | How to install |
|---|---|---|
| macOS | **Yes** (for menu bar; Linux works for server + browser UI) | — |
| Python ≥ 3.11 | **Yes** | [uv](https://docs.astral.sh/uv/) recommended (`server.py` has PEP 723 inline deps) |
| [tmux](https://github.com/tmux/tmux) ≥ 3.0 | **Yes** — for in-browser terminals | `brew install tmux` |
| [Claude Code](https://claude.com/claude-code) | **Yes** | — |
| [Hammerspoon](https://www.hammerspoon.org) | Optional — for menu bar tray | `brew install --cask hammerspoon` |
| `ANTHROPIC_API_KEY` env var | Optional — enables rolling summaries | set in your shell |

External sessions (started from your own terminal, no tmux) appear in the list read-only. Only tmux-backed sessions (created via `+ New instance`) get the in-browser terminal.

## Install

```sh
git clone https://github.com/lmarini556/claude-instances-ui.git
cd claude-instances-ui

# 1. Install Claude Code hooks (idempotent; copies hook_writer.py to a stable location)
python3 install_hooks.py

# 2. Make sure tmux is installed
brew install tmux

# 3. Run the server
uv run server.py
#   or:  pip install fastapi 'uvicorn[standard]' httpx websockets && python3 server.py
```

On first startup the server prints something like:

```
========================================================================
  Claude Instances UI
========================================================================
  Dashboard:  http://127.0.0.1:7878
  Authenticate (first visit): http://127.0.0.1:7878/auth?t=xk4B7... (long token)
  Token file: /Users/you/.claude-instances-ui/token
  tmux:       available
========================================================================
```

Click the `/auth?t=...` URL once to set an auth cookie. After that, open [http://127.0.0.1:7878](http://127.0.0.1:7878) directly.

### Optional: Hammerspoon menu bar tray

Install [Hammerspoon](https://www.hammerspoon.org), then copy the Lua module into your config:

```sh
mkdir -p ~/.hammerspoon
cp menubar/claude_instances.lua ~/.hammerspoon/claude_instances.lua
grep -qxF 'require("claude_instances")' ~/.hammerspoon/init.lua 2>/dev/null \
  || echo 'require("claude_instances")' >> ~/.hammerspoon/init.lua
```

Reload Hammerspoon (menu bar → Reload Config). A `◉` icon appears; click or press `⌘⇧C` for the slide-in tray. The tray reads your auth token from `~/.claude-instances-ui/token` automatically.

> Re-run the `cp` line (and re-run `install_hooks.py`) after `git pull` to pick up updates.

## Configuration

All env vars optional. Defaults in **bold**.

| Variable | Default | Purpose |
|---|---|---|
| `CIU_HOST` | **`127.0.0.1`** | FastAPI bind host |
| `CIU_PORT` | **`7878`** | FastAPI port |
| `CIU_PUBLIC_URL` | **`http://$CIU_HOST:$CIU_PORT`** | Canonical URL (override if reverse-proxying) |
| `ANTHROPIC_API_KEY` | _unset_ | Enables API-backed summaries |
| `CLAUDE_INSTANCES_SUMMARY_MODEL` | **`claude-haiku-4-5`** | Summarizer model |
| `CLAUDE_INSTANCES_UI_EPHEMERAL` | _unset_ | Internal marker — disables hook state writes (used by summarizer's own `claude -p` calls) |
| `CLAUDE_INSTANCES_UI_OWNED` | _auto_ | Stamped on tmux-spawned sessions by the server; hooks persist it into state files |

## Security

The server exposes endpoints that can:

- Spawn arbitrary processes in arbitrary directories (`POST /api/instances/new`)
- Inject keystrokes into running sessions (websocket)
- Send signals to running processes

**Do not expose the server to an untrusted network without the auth token.** The token is mandatory on every `/api/*` and `/ws/*` endpoint. Setting `CIU_HOST=0.0.0.0` makes the server reachable on your LAN — only do this behind a trusted network (Tailscale, home LAN) and treat the token as the sole barrier.

To rotate the token: delete `~/.claude-instances-ui/token` and restart the server.

## State layout

Everything under `~/.claude-instances-ui/`:

```
~/.claude-instances-ui/
├── bin/hook_writer.py     # installed copy of the hook writer (stable path)
├── state/<sid>.json       # per-session state, written by hooks
├── summary/<sid>.json     # rolling summaries from the summarizer
├── tmux/<our_sid>.fifo    # pipe-pane FIFOs for each tmux-owned session
├── groups.json            # user-defined groupings
├── names.json             # custom names
├── acks.json              # "reply acknowledged" timestamps
├── menubar_acks.json      # Hammerspoon-side dismissal state
├── token                  # auth token (mode 600)
├── hook.log               # hook writer errors
└── summarizer.log         # summarizer errors
```

To fully reset: stop the server, then `rm -rf ~/.claude-instances-ui/`.

## Limitations

- **Menu-bar tray is macOS only** (Hammerspoon-based). The server and web UI work on Linux.
- **tmux is required** for the in-browser terminal feature. External sessions degrade gracefully to read-only transcript view.
- **No session persistence across host reboots** in v0 — tmux server dies with the host. Sessions started via `claude` directly behave identically to before.
- **`install_hooks.py` writes `~/.claude/settings.json` without a backup.** It's idempotent and marker-tagged, but back it up yourself if you have unusual content.
- **HTTPS not built-in.** Terminate TLS in a reverse proxy if you need it.
